# Project Guidelines: Google Extensions - Project Manager

## Coding Standards
- Formatting: No redundant blank lines. No blank lines at the end of files.
- JS control flow: `if`, `else`, `for`, `while` must always use `{}`, even for single-line bodies. The body must be on a new line, never on the same line as the opening `{`.
- HTML inline styles: property-value pairs must have a space after `:` and after `;`. Example: `style="display: none; margin-top: 8px;"`

## Autonomy
- Do not hesitate to ask the user for clarification or confirmation whenever uncertain — proactive communication is encouraged.
- For major changes that affect API contracts, data structures, or architectural design, always ask the user before proceeding.

## Restrictions
- Git: DO NOT execute any Git commands (add, commit, push, etc.) without explicit user permission.
